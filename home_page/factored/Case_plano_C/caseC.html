Case Completo: Projeto de Organização de Atendimentos (NPJ)

Contexto e Motivação

A necessidade do projeto surgiu diante do alto volume de atendimentos e dos frequentes casos de extravio de documentação durante as etapas procedimentais internas da instituição. Havia demanda por um sistema que preservasse a integridade dos registros gerados pelos alunos atendentes do Núcleo de Prática Jurídica.

Objetivo

Criar um programa robusto para armazenar e organizar os dados coletados pelos alunos do NPJ, permitindo que orientadores acessem, filtrem e realizem acompanhamento dos atendimentos sem perda de informação.

Tecnologias e Arquitetura

  • Back-end e lógica principal: Java (aplicação desktop)
  • Interface: FXML com estilização em CSS para visão clara e intuitiva
  • Armazenamento: base local/centralizada (pode ser adaptada para SQLite, H2 ou integração com servidor)
  • Usuários: alunos (atendentes) com perfis de cadastro; orientadores com permissões de revisão

Funcionalidades Principais

  • Formulário de atendimento com campos validados para captura padronizada de dados
  • Armazenamento seguro dos registros para evitar extravio
  • Pesquisa, filtros e exportação básica de dados para análise pelos orientadores
  • Registro de histórico e trilhas de auditoria das alterações
  • Interface intuitiva em FXML pensada para uso por estudantes

Impacto e Resultados Esperados

Com a adoção do sistema, espera-se reduzir perdas documentais, agilizar o fluxo de atendimento e melhorar a qualidade dos registros entregues aos orientadores, tornando mais eficiente o acompanhamento dos casos e a geração de relatórios acadêmicos e administrativos.

Observações Técnicas

O projeto foi codificado majoritariamente em Java com uso de FXML para separar apresentação e lógica. A arquitetura permite futuras integrações (API REST, sincronização com servidor, autenticação centralizada) conforme a maturidade e demanda da instituição.
